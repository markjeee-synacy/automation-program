# Knowledge Base (`docs/kb/`)

Internal Markdown knowledge base for the RISE Automation Program. Records program context, decisions, research notes, and reference material that informs program-management work.

## Status

Internal-only. Not published to the Astro site, Confluence, or Google Docs. The Publishing Pipeline reads from `program-files/`, never from `docs/kb/`.

## Scope

KB pages may describe any class defined in `WORLDVIEW.md` (Opportunities, Initiatives, Objectives, KPIs, Roadmap items, Working Groups, Work Boards, Collaboration Channels, Processes, Roles), as well as supporting research, decision records, and background material that does not belong in the program-of-record under `program-files/`.

## Placement guidance

- Use `program-files/` for program-of-record artifacts that should be published or co-edited (Mission, Sub-Programs, Internal Programs, Initiative Documents, Roadmap Documents, Strategy Documents, work-system documents).
- Use `docs/kb/` for internal context that supports those artifacts but is not itself program-of-record.
- See `WORLDVIEW.md` (KB Page, KB Directory) and `RULES.md` (Knowledge Base Edits) for the authoritative definitions and edit rules.

## Edit rules

Edits here are internal and have no publishing side-effects. Bulk renames or deletions still warrant developer confirmation because internal cross-references (and `AGENTS.md` / `WORLDVIEW.md` citations) may break.
