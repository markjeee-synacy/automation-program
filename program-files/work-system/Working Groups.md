# Working Groups

## Overview

Working groups are long-lived virtual teams, formed to work together around system domains.

Since initiatives in programs may affect work across system domains, working groups are not exclusive to initiatives, and most work across – depending on the domains affected by an idea.

Working Groups is Automation program's resource deployment strategy. It is meant to be long-lived, and independent of Programs and Initiatives, though closely track, constraint by the learning curve and need for depth in domain knowledge.
