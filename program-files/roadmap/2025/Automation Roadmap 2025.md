### Quality Automation

These are items aimed to improve and enhance the quality of what we have in place.

| **Problem / Opportunity** | **Solution / Idea** | **Timeline** |
| --- | --- | --- |
| **Spill over from 2024** |  |  |
| **Improved Deal metadata** _Initiative: Sales IS_ | **Proposed** Re-implement Deal Metadata calculations using an external Java app _(from HS workflows)_ Re-implement HubSpot → Airtable sync using an external Java app _(from Make)_ | 2025 H1 or H2, pending software team availability |
| **New from 2025** |  |  |
| Enhance quality of Site records _Initiative: TBD in Framework_ | **Proposed** Improved definition of Sites and Areas, introduced a unique subject identification number Transition Site records used in areas to use Area records Created Site and Area records for core network infrastructure | 2025 H1 |
| Enhance quality of Customer and Service records _Initiative: TBD in Framework_ | **Proposed** Introduced a unique subject identification number Standard site relation references _(e.g. Site A points to customer Site, Site B to data center)_ Specific complimentary tags _(e.g. acquisition, bundled, perks)_ | 2025 H1 |
| Migrate Aquisition Records to HubSpot _Initiative: Sales IS_ | Deal Won in SAQ pipeline triggers the creation of an Acquisition Record in HubSpot SAQ manages the AR lifecycle in HubSpot | Need opportunity development, aimed to start 2025 Q3 |
